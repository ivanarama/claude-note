# Claude Code запоминает всё: как перестать терять знания между сессиями

## Проблема, которую знает каждый

Вчера вы час объясняли Claude структуру обработки обмена УТ11 → БП3: «файл Module.bsl, кодировка UTF-8 with BOM, в regex для отступов используй `[ \t]*`, а не `\s*`, потому что `\s` ловит переносы строк...»

Сегодня открыли новую сессию — и Claude снова ничего не знает. Начинаете сначала.

Знакомо?

Если вы работаете с Claude Code больше недели, у вас наверняка есть:

- Файл `промпты.txt` (или `промпты_v3_final_FINAL.txt`)
- `CLAUDE.md` в проекте, куда вы руками записываете правила, чтобы Claude не повторял ошибки
- Десяток заметок в блокноте, разбросанных по разным папкам
- Ощущение, что половину рабочего времени тратите на передачу контекста, а не на работу

CLAUDE.md — отличный инструмент, но он требует дисциплины. Нужно самому следить за тем, чтобы после каждой сессии записать, что было сделано, какие решения приняли, какие ошибки допустили. Забыл записать — потерял.

## Что если бы это происходило автоматически?

**claude-note** — open-source проект, который тихо работает в фоне и автоматически превращает ваши сессии с Claude Code в структурированные заметки. Без ручного труда. Без копирования промптов. Без файлов `v3_final`.

Схема простая:

```
Вы работаете в Claude Code как обычно
              ↓
    claude-note тихо слушает
              ↓
    Сессия закончилась
              ↓
    Claude анализирует транскрипт
              ↓
    В вашем Obsidian появляется заметка:
      - Что делали
      - Какие решения приняли
      - Какие ошибки допустили
      - Какие вопросы остались
```

Никакой магии — просто хуки Claude Code + фоновый воркер + вызов Claude для синтеза знаний.

## Реальные примеры: как это помогает 1С-разработчику

### Пример 1: Рефакторинг обработки обмена данных

Допустим, вы рефакторите обработку переноса данных УТ11 → БП3. Файл на 20 000 строк, 48 блоков с однотипной логикой проведения документов. Claude помогает писать regex-скрипты для массовой замены.

В процессе работы вы натыкаетесь на кучу подводных камней. С claude-note всё это автоматически сохраняется:

```markdown
### Концепция: Regex в коде 1С — подводные камни

При рефакторинге BSL-кода регулярными выражениями критично
учитывать особенности формата:

- `\s*` ловит переносы строк — для отступов использовать `[ \t]*`
- Пустые строки между операторами ломают построчный regex —
  вставлять `(?:[ \t]*\r?\n)*` между строками паттерна
- Windows-переносы `\r\n` — во всех regex использовать `\r?\n`
- `[^,]+?` не работает с вложенными вызовами функций —
  нужен парсер баланса скобок

### Решение: Порядок применения скриптов рефакторинга
При замене больших блоков по номерам строк —
применять снизу вверх, чтобы номера строк не сдвигались.

### Gotcha: Самозамена определения функции
Regex-скрипт нашёл совпадение внутри самого определения
новой функции-помощника и заменил тело функции на вызов
самой себя. Правило: всегда пропускать секцию определений
вспомогательных функций.
```

Через месяц вам достаётся похожая обработка обмена. Вы открываете Obsidian — а там вся карта грабель, на которые вы уже наступали. Не надо заново набивать шишки.

### Пример 2: Оптимизация запроса отчёта

Отчёт «Валовая прибыль» грузится 4 минуты. Вы с Claude анализируете план запроса, перестраиваете СКД. claude-note фиксирует:

```markdown
### Концепция: Оптимизация запросов СКД

Основное узкое место — отбор по периоду в WHERE вместо
параметров виртуальной таблицы. СКД генерирует запрос
без условий внутри ВТ, и СУБД сканирует весь регистр.

### How-to: Ускорение отчёта по регистру накопления
1. Перенести отбор по периоду в параметры виртуальной таблицы
   (было в WHERE, стало в параметрах ВТ)
2. Добавить индекс (Номенклатура + Период) на регистр
3. Не использовать ОБЪЕДИНИТЬ ВСЕ с виртуальными таблицами
   разных регистров — оптимизатор не использует индексы
4. Результат: 4 мин → 12 сек

### Открытый вопрос
Нужно ли создавать отдельный регистр-агрегат для этого
отчёта, или достаточно индекса? Обсудить с аналитиком.
```

### Пример 3: Разбор чужого обмена

Достался проект с обменом между УТ и БП через план обмена. 15 правил конвертации, документации ноль. Вы работаете с Claude несколько дней, по 2-3 сессии в день.

Без claude-note: каждую сессию начинаете с «у нас план обмена ОбменУТБП, кастомные правила, вот структура...»

С claude-note: знания накапливаются автоматически. Через неделю в вашем Obsidian — полная карта обмена:

```markdown
### Концепция: Обмен УТ ↔ БП — архитектура
- План обмена «ОбменУТБП» — узел БП регистрируется по организации
- 8 из 15 правил конвертации кастомные (не типовые)
- Документ «Реализация» конвертируется через промежуточную
  структуру из-за различий в табличных частях

### How-to: Отладка правила конвертации ПКО №7
1. Точка останова в модуле «ПравилаОбмена» строка 847
2. Смотреть переменную «ТаблицаСоответствий»
3. НДС передавался в копейках, а БП ожидает в рублях

### Решение: Дубли при повторном обмене
Причина: не было фильтра по дате запрета.
Исправление: добавили отбор в запрос узла обмена.
```

### Пример 4: Архив промптов — библиотека приёмов

С включённым архивом промптов claude-note сохраняет все ваши запросы:

```markdown
### 2025-03-15 09:30 — рефакторинг обработки обмена
1. Покажи структуру файла Module.bsl, найди все блоки
   с паттерном НайтиПоНомеру
2. Напиши regex-скрипт для замены ПодготовитьМассивИНН
   на вызов общей функции
3. Почему regex съел определение самой функции? Исправь,
   добавь пропуск секции определений
4. Теперь сделай то же самое для ОпределитьДействиеСДокументом,
   учитывая что блоки ПКО/РКО используют условный выбор типа

### 2025-03-16 14:00 — оптимизация отчёта
1. Вот запрос СКД, объясни план выполнения
2. Почему виртуальная таблица Остатки сканирует весь регистр?
3. Перепиши с отбором в параметрах ВТ
```

Через полгода вы видите, какие промпты работали хорошо, а какие приводили в тупик. Можно переиспользовать формулировки.

## Как CLAUDE.md и claude-note дополняют друг друга

CLAUDE.md — это ваши **правила для Claude**: «делай так, не делай так». Вы пишете его вручную и осознанно.

claude-note — это **автоматическая память**: что произошло, какие решения были приняты, какие ошибки допущены. Пишется сама.

Они не конкурируют, а дополняют друг друга:

| | CLAUDE.md | claude-note |
|---|-----------|-------------|
| Кто пишет | Вы, вручную | Автоматически |
| Что хранит | Правила, конвенции | Факты, решения, вопросы |
| Где живёт | В корне проекта | В Obsidian vault |
| Когда полезно | В начале сессии | После сессии и через неделю |
| Пример | «В regex используй `[ \t]*`» | «15.03 regex `\s*` сломал замену, переделали на `[ \t]*`» |

Лучшая стратегия: claude-note фиксирует всё автоматически, а вы периодически просматриваете заметки и самое важное переносите в CLAUDE.md проекта.

## Установка на Windows

Большинство 1С-разработчиков работают на Windows. claude-note поддерживает Windows 10/11. Вот пошаговая инструкция.

### Что понадобится

- **Python 3.11+** — скачайте с [python.org](https://www.python.org/downloads/), при установке поставьте галку «Add to PATH»
- **Git** — скачайте с [git-scm.com](https://git-scm.com/download/win)
- **Claude Code** (Claude CLI) — нужен для синтеза знаний. Установите по [инструкции](https://docs.anthropic.com/en/docs/claude-code/overview)
- **Obsidian** — бесплатная версия, скачайте с [obsidian.md](https://obsidian.md). Или любая папка для markdown-заметок

### Шаг 1: Клонирование и установка

Откройте PowerShell и выполните:

```powershell
# Склонировать репозиторий
git clone https://github.com/ivanarama/claude-note.git
cd claude-note

# Установить пакет
pip install -e .
```

Флаг `-e` (editable) означает, что пакет устанавливается «по ссылке» — вы сможете обновлять его через `git pull` без переустановки.

### Шаг 2: Создание конфигурации

```powershell
# Создать папку конфигурации
mkdir $env:USERPROFILE\.config\claude-note
```

Создайте файл `%USERPROFILE%\.config\claude-note\config.toml` с содержимым:

```toml
# Путь к вашему Obsidian vault (или любой папке для заметок)
vault_root = "C:\\Users\\ВашеИмя\\Documents\\MyVault"

[synthesis]
mode = "route"                          # route = полный режим с созданием заметок
model = "claude-sonnet-4-5-20250929"    # модель для синтеза

[language]
code = "ru"          # Заметки и промпты синтеза на русском

[prompts_archive]
enabled = true       # Сохранять все ваши промпты
```

Замените путь на реальный путь к вашему vault. Обратите внимание на двойные обратные слеши.

### Шаг 3: Настройка хуков Claude Code

Хуки — это то, как Claude Code уведомляет claude-note о событиях. Откройте файл `%USERPROFILE%\.claude\settings.json` и добавьте:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python -m claude_note enqueue",
            "timeout": 5000
          }
        ]
      }
    ],
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python -m claude_note enqueue",
            "timeout": 5000
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python -m claude_note enqueue",
            "timeout": 5000
          }
        ]
      }
    ]
  }
}
```

> Если у вас уже есть `settings.json` с другими настройками — добавьте секцию `hooks` к существующим, не перезаписывая файл целиком.

### Шаг 4: Запуск воркера

Воркер — фоновый процесс, который обрабатывает события и генерирует заметки.

Для проверки запустите его в режиме переднего плана:

```powershell
python -m claude_note worker --foreground --verbose
```

Вы увидите:
```
2025-03-18 10:00:00 [INFO] Worker starting
```

Теперь откройте другое окно терминала, запустите Claude Code и поработайте немного. В логах воркера вы увидите, как обрабатываются события.

### Шаг 5: Фоновый запуск

Для постоянной работы запустите воркер в фоне:

```powershell
Start-Process python -ArgumentList "-m claude_note worker" -WindowStyle Hidden
```

Или создайте задачу в Планировщике Windows (Task Scheduler), чтобы воркер стартовал автоматически при входе в систему:

1. Откройте Планировщик заданий
2. Создать задачу → Имя: `claude-note`
3. Триггер: «При входе в систему»
4. Действие: Запуск программы
   - Программа: `python`
   - Аргументы: `-m claude_note worker`
5. Снять галку «Останавливать задачу, выполняемую дольше...»

### Шаг 6: Проверка

```powershell
python -m claude_note status
```

Должны увидеть статус воркера и очереди. Откройте Obsidian — после первой завершённой сессии Claude Code в vault появится файл `claude-note-inbox.md` с извлечёнными знаниями.

## Русский язык

claude-note поддерживает русский язык. С настройкой `language.code = "ru"` все промпты синтеза, описания схем и лейблы интерфейса будут на русском. Заметки в Obsidian тоже будут генерироваться на русском.
